SPI_Init

描述：

初始化配置USB2SPI适配器。

原型：

  1. int WINAPI SPI_Init(int DevHandle,int SPIIndex, PSPI_CONFIG pConfig);  

参数：


DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

SPIIndex  SPI通道号，从机模式下只能用SPI1_CS0或SPI2_CS0，主机模式下可以使用SPI1_CS0，SPI1_CS1，SPI1_CS2，SPI1_CS3，SPI1_CS4，SPI2_CS0，SPI2_CS1，SPI2_CS2，SPI2_CS3，SPI2_CS4.

pConfig  初始化配置结构体指针，具体取值说明见文章开头的说明。

返回值：

函数执行状态，0为成功，其他值为失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2spi.h"  
  3.     
  4. int ret;  
  5. int SPIIndex = SPI1_CS0;  
  6. //配置SPI总线相关参数   
  7. SPIConfig.Mode = 0;  
  8. SPIConfig.ClockSpeedHz = 25000000;  
  9. SPIConfig.CPHA = 0;  
  10. SPIConfig.CPOL = 0;  
  11. SPIConfig.LSBFirst = 0;  
  12. SPIConfig.Master = 1;  
  13. SPIConfig.SelPolarity = 0;  
  14. ret = SPI_Init(DevHandle,SPIIndex,&SPIConfig);  
  15. if(ret != SPI_SUCCESS){  
  16.     printf("初始化配置USB2SPI适配器失败!\n");  
  17.     return;  
  18. }  

SPI_WriteBytes

描述：

主机模式下SPI发送数据，该函数必须等到SPI数据发送完毕之后才会返回，对于SPI时钟频率较低，单次发送的数据量较大的情况，函数执行时间可能会比较长。

原型：

  1. int WINAPI SPI_WriteBytes(int DevHandle,int SPIIndex,unsigned char *pWriteData,int WriteLen);  

参数：

DevHandle   设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

SPIIndex   SPI通道号，可取以下值：SPI1_CS0，SPI1_CS1，SPI1_CS2，SPI1_CS3，SPI1_CS4，SPI2_CS0，SPI2_CS1，SPI2_CS2，SPI2_CS3，SPI2_CS4.

pWriteData   待发送的数据缓冲区指针。

WriteLen   待发送的数据字节数，单次发送的数据不能超过20*1024字节。

返回值：

函数执行状态，0为成功，其他值为失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2spi.h"  
  3.     
  4. int ret;  
  5. int SPIIndex = SPI1_CS0;  
  6. unsigned char WriteBuffer[20*1024];  
  7. //SPI发送数据     
  8. for(int i=0;i<sizeof(WriteBuffer);i++){  
  9.     WriteBuffer[i] = i;  
  10. }  
  11. ret = SPI_WriteBytes(DevHandle,SPIIndex,WriteBuffer,sizeof(WriteBuffer));  
  12. if(ret != SPI_SUCCESS){  
  13.     printf("SPI发送数据失败!\n");  
  14.     return;  
  15. }  

SPI_WriteBytesAsync

描述：

主机异步模式下发送数据，调用该函数后，数据会传递到适配器内部数据缓冲区中，然后启动DMA开始数据传输，函数返回。下一次再次调用该函数的时候，数据会发送到适配器内部数据缓冲区，然后先判断上一次的数据是否发送完毕，若已经发送完毕则再次启动DMA开始数据传输，函数返回。

使用该函数可以最大速度输出SPI数据，适配器接收数据是采用的双缓区方式，所以不用担心数据被覆盖。

原型：

  1. int WINAPI SPI_WriteBytesAsync(int DevHandle,int SPIIndex,unsigned char *pWriteData,int WriteLen);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

SPIIndex  SPI通道号，可取以下值：SPI1_CS0，SPI1_CS1，SPI1_CS2，SPI1_CS3，SPI1_CS4，SPI2_CS0，SPI2_CS1，SPI2_CS2，SPI2_CS3，SPI2_CS4.

pWriteData  待发送的数据缓冲区指针。

WriteLen  待发送的数据字节数，单次发送的数据不能超过20*1024字节。

返回值：

函数执行状态，0为成功，其他值为失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2spi.h"  
  3.     
  4. int ret;  
  5. int SPIIndex = SPI1_CS0;  
  6. unsigned char WriteBuffer[20*1024];  
  7. //SPI发送数据     
  8. for(int i=0;i<sizeof(WriteBuffer);i++){  
  9.     WriteBuffer[i] = i;  
  10. }  
  11. ret = SPI_WriteBytesAsync(DevHandle,SPIIndex,WriteBuffer,sizeof(WriteBuffer));  
  12. if(ret != SPI_SUCCESS){  
  13.     printf("SPI发送数据失败!\n");  
  14.     return;  
  15. }

原型：

  1. int WINAPI SPI_BlockReadBytesOfEvent(int DevHandle,int SPIIndex,unsigned char *pReadData,int BlockSize,int BlockNum,int EventPin,unsigned char EventType,int TimeOutOfMs);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

SPIIndex  SPI通道号，可取以下值：SPI1_CS0，SPI1_CS1，SPI1_CS2，SPI1_CS3，SPI1_CS4，SPI2_CS0，SPI2_CS1，SPI2_CS2，SPI2_CS3，SPI2_CS4.

pReadData  读数据缓冲区首地址。

BlockSize  每次读数据大小，单位为字节。

BlockNum  读数据次数，块大小和块数乘积不能大于10K。

EventPin  等待事件输入的GPIO引脚，每个bit位对应一个引脚，比如P0为0x01，P2为0x04，P15为0x8000。

EventType   等待件类型，0x00-低电平，0x11-高电平，0x10-下降沿，0x01上升沿。

TimeOutOfMs  等待事件超时时间，单位为毫秒，等待超时后该函数直接返回，不执行SPI任何操作。

返回值：

函数执行状态，0为成功，其他值为失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2spi.h"  
  3.     
  4. int ret;  
  5. int SPIIndex = SPI1_CS0;  
  6. unsigned char ReadBuffer[20*1024];  
  7. int BlockSize = 10;  
  8. int BlockNum = 30;  
  9. int i = 0,j = 0;  
  10. int IntervalTimeUs = 20;  
  11. //读BlockNum次数据，每次读BlockSize字节数据，每次读数据都必须等待P0引脚的下降沿事件，每次读数据之间，片选信号会自动无效  
  12. ret = SPI_BlockReadBytesOfEvent(DevHandle,SPIIndex,ReadBuffer,BlockSize,BlockNum,0x01,0x10,100);  
  13. if(ret != SPI_SUCCESS){  
  14.     printf("Block读数据出错!\n");  
  15.     return;  
  16. }  

SPI_BlockWriteReadBytesOfEvent

描述：

事件Block模式写读数据，也就是将SPI_WriteReadBytesOfEvent多次操作合并为一个操作。

原型：

  1. int WINAPI SPI_BlockWriteReadBytesOfEvent(int DevHandle,int SPIIndex,unsigned char *pWriteData,int WriteBlockSize,unsigned char *pReadData,int ReadBlockSize,int BlockNum,int EventPin,unsigned char EventType,int TimeOutOfMs);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

SPIIndex  SPI通道号，可取以下值：SPI1_CS0，SPI1_CS1，SPI1_CS2，SPI1_CS3，SPI1_CS4，SPI2_CS0，SPI2_CS1，SPI2_CS2，SPI2_CS3，SPI2_CS4.

pWriteData  写数据缓冲区首地址。

WriteBlockSize  每次写数据大小，单位为字节。

pReadData  读数据缓冲区首地址。

ReadBlockSize  每次读数据大小，单位为字节。

BlockNum  写读数据次数，块大小和块数乘积不能大于10K。

EventPin  等待事件输入的GPIO引脚，每个bit位对应一个引脚，比如P0为0x01，P2为0x04，P15为0x8000。

EventType   等待件类型，0x00-低电平，0x11-高电平，0x10-下降沿，0x01上升沿。

TimeOutOfMs  等待事件超时时间，单位为毫秒，等待超时后该函数直接返回，不执行SPI任何操作。

返回值：

函数执行状态，0为成功，其他值为失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2spi.h"  
  3.     
  4. int ret;  
  5. int SPIIndex = SPI1_CS0;  
  6. unsigned char WriteBuffer[20*1024];  
  7. unsigned char ReadBuffer[20*1024];  
  8. int WriteBlockSize = 10;  
  9. int ReadBlockSize = 2;  
  10. int BlockNum = 30;  
  11. int i = 0,j = 0;  
  12. int IntervalTimeUs = 20;  
  13. //准备数据，SPI写读BlockNum次数据，每次写WriteBlockSize字节数据然后读ReadBlockSize字节数据，每次写读数据时必须等待P0引脚的下降沿事件，每次写读数据之间，片选信号会自动无效  
  14. for(i=0;i<BlockNum;i++){  
  15.     for(j=0;j<BlockSize;j++){  
  16.         WriteBuffer[i*BlockSize+j] = j;  
  17.     }  
  18. }  
  19. ret = SPI_BlockWriteReadBytesOfEvent(DevHandle,SPIIndex,WriteBuffer,WriteBlockSize,ReadBuffer,ReadBlockSize,BlockNum,0x01,0x10,100);  
  20. if(ret != SPI_SUCCESS){  
  21.     printf("Block写读数据出错!\n");  
  22.     return;  
  23. }
