概述

本文主要介绍兼容ZLG的CAN卡的接口函数。若之前有使用ZLG的接口函数开发应用软件，或者是想使用ZLG的CANTest和CANPro软件，那么只需要将我们的USB2XXX.dll和libusb-1.0.dll文件拷贝到ControlCAN.dll文件所在目录，然后将USB2XXX.dll文件重新命名为ControlCAN.dll，并覆盖原来的文件即可。

VCI_ScanDevice

描述：

扫描当前连接到电脑端的USB2XXX设备，改函数可以不用调用。

原型：

  1. unsigned int WINAPI VCI_ScanDevice(unsigned char NeedInit);  

参数：

NeedInit 保留参数，可以传入0.

返回值：

当前连接到电脑的设备数。

VCI_OpenDevice

描述：

打开设备，该函数必须调用。

原型：

  1. unsigned int WINAPI VCI_OpenDevice(unsigned int DevType,unsigned int DevIndex,unsigned int Reserved);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

Reserved 保留，若是安卓系统则传入设备句柄，具体请参考我们提供的范例程序源码。

返回值：

0-函数调用出错，1-函数调用成功。

VCI_CloseDevice

描述：

关闭已经被打开的设备，若已经启动了CAN，则会自动停止CAN。

原型：

  1. unsigned int WINAPI VCI_CloseDevice(unsigned int DevType,unsigned int DevIndex);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

返回值：

0-函数调用出错，1-函数调用成功。

VCI_InitCAN

描述：

初始化CAN，必须调用。

原型：

  1. unsigned int WINAPI VCI_InitCAN(unsigned int DevType, unsigned int DevIndex, unsigned int CANIndex, PVCI_INIT_CONFIG pInitConfig);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

pInitConfig 初始化参数结构体指针。

返回值：

0-函数调用出错，1-函数调用成功。

VCI_GetReceiveNum

描述：

获取当前CAN接收数据缓冲区中有效帧数。

原型：

  1. unsigned int WINAPI VCI_GetReceiveNum(unsigned int DevType,unsigned int DevIndex,unsigned int CANIndex);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

返回值：

CAN数据接收缓冲区中的CAN帧数。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

返回值：

0-函数调用出错，1-函数调用成功。

VCI_StartCAN

描述：

启动CAN，调用该函数后，底层会自动开启一个接收CAN数据的线程，接收到CAN数据之后自动放入内部数据缓冲区中。

原型：

  1. unsigned int WINAPI VCI_StartCAN(unsigned int DevType,unsigned int DevIndex,unsigned int CANIndex);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

返回值：

0-函数调用出错，1-函数调用成功。

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

返回值：

0-函数调用出错，1-函数调用成功。

VCI_Transmit

描述：

发送CAN数据。

原型：

  1. unsigned int WINAPI VCI_Transmit(unsigned int DevType,unsigned int DevIndex,unsigned int CANIndex,PVCI_CAN_OBJ pSend,unsigned int Len);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

pSend 待发送的CAN数据缓冲区指针。

Len 待发送的CAN数据长度。

返回值：

成功发送的CAN帧数。

VCI_Receive

描述：

从CAN数据缓冲区中获取CAN数据。

原型：

  1. unsigned int WINAPI VCI_Receive(unsigned int DevType,unsigned int DevIndex,unsigned int CANIndex,PVCI_CAN_OBJ pReceive,unsigned int Len,int WaitTime=-1);  

参数：

DevType 设备类型，若设备只有一个CAN通道，选择VCI_USBCAN1，若设备有2个CAN通道，则选择VCI_USBCAN2。

DevIndex 设备索引号，若只连接了一个设备，则传入0即可。

CANIndex CAN通道索引，0-对应CAN1通道，1-对应CAN2通道。

pReceive 存储CAN数据的缓冲区指针。

Len 存储CAN数据的缓冲区大小。

WaitTime 保留。

返回值：

实际获取到的CAN数据帧数。
