概述

本文介绍了IIC/I2C总线监控接口函数，利用这些函数，可以自己写上位机软件实现对IIC/I2C总线的数据监控，存储，分析，处理。

下面是本文函数可能会用到的一些宏定义：

  7.     DATA,  
  8.     STOP  
  9. }I2C_DATA_TYPE;  
  10. //I2C数据结构定义  
  11. typedef struct  
  12. {

  14.     unsigned char ACK;          //0-当前ACK为低电平，1-当前ACK为高电平  
  15.     unsigned char WriteRead;//1-读操作，0-写操作  
  16.     unsigned char Data;         //当DataType为DATA时对应的数据  
  17.     unsigned short Addr;        //数据对应的从机地址

I2C_SnifferStart

描述：

启动I2C总线监控功能，启动的时候需要传入一个回调函数，抓取到数据后会自动调用回调函数实现数据输出。

原型：

  1. int WINAPI I2C_SnifferStart(int DevHandle,int Channel,unsigned int SampleRateHz,I2C_GET_DATA_HANDLE *pGetI2CDataHandle);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

Channel  监控通道号，0-对应D0(SCL)和D1(SDA)，1-对应D2(SCL)和D3(SDA)，2-对应D4(SCL)和D5(SDA)，3-对应D6(SCL)和D7(SDA)。

SampleRateHz 采样波特率，建议设置为5000000或者2500000。

pGetI2CDataHandle 数据输出回调函数，函数原型见文章开头部分宏定义。

返回值：

0函数调用成功，启动I2C监控功能成功，其他值，函数执行失败，启动I2C监控失败。

I2C_SnifferStop

描述：

停止I2C监控功能。

原型：

  1. int WINAPI I2C_SnifferStop(int DevHandle);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

返回值：

0停止失败，其他值函数调用失败，停止I2C监控失败。
