概述 

本文主要介绍关于USB2XXX的并口操作相关函数，也就是读写控制D0到D7接口的函数，利用并口操作函数可以快速的读写控制并口，并口最大读写速度可达25MBye/s。

在写数据的时候，CE和WE会自动变低电平，数据写完毕之后自动变高电平，在读数据的时候，CE和RE自动变低电平，数据读完毕之后自动变高电平。

在调用这些函数前需要调用设备操作相关函数，比如扫描设备，打开设备等，详情请查看《设备操作相关函数说明》。

  4. #define SNIFFER_ERR_USB_WRITE_FAIL   (-2)  //USB写数据失败  
  5. #define SNIFFER_ERR_USB_READ_FAIL     (-3)  //USB读数据失败  
  6. #define SNIFFER_ERR_CMD_FAIL                (-4)  //命令执行失败

  8.

SNIFFER_Init

描述：

初始化配置并口，配置其读写速度，读写模式，采样模式等。

原型：

  1. int WINAPI SNIFFER_Init(int DevHandle,char WriteFlag,unsigned int SampleRateHz,char SampleMode);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

WriteFlag  配置并口是都操作还是写操作，0-读操作，1-写操作。

SampleRateHz  读写操作并口的频率，单位为Hz，只能在以下几个值间进行选择：25000000，20000000，10000000，5000000，2500000，1000000，700000。

SampleMode  配置并口采集位数，1-1bit，2-2bit，4-4bit，8-8bit，一般传入8即可，也就是采集8bit，1字节数据，该参数只有在读操作时有效。

返回值：

配置并口状态，0-配置成功，其他值配置失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2sniffer.h"  
  3.    
  4. int ret;  
  5.    
  6. //配置并口为读数据模式,每次采集1字节数据，数据读取速率为5MByte/s  
  7. ret = SNIFFER_Init(DevHandle,0,5000000,8);  
  8. if(ret != SNIFFER_SUCCESS){  
  9.  printf("初始化配置并口失败!\n");  
  10.  return;  
  11. }  

SNIFFER_StartRead

描述：

启动并口连续读取数据，读取到的数据通过回调函数方式传出。

原型：

  1. int WINAPI SNIFFER_StartRead(int DevHandle,SNIFFER_GET_DATA_HANDLE *pGetDataHandle);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

pGetDataHandle  读到数据后的回掉函数，必须传入有效的回调函数，否则无法获取到并口读到的数据。

返回值：

启动并口读数据状态，0-启动成功，其他值，启动失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2sniffer.h"  
  3.    
  4. //此处为接收数据回调函数，在这里处理接收到的数据  
  5. void SnifferGetData(int DeviceIndex,unsigned char *pData,int DataNum)  
  6. {  
  7.     printf("获取到%d字节数据\n",DataNum);  
  8. }  
  9.    
  10. int ret;  
  11.    
  12. //启动sniffer并注册回调函数  
  13. ret = SNIFFER_StartRead(DevHandle,SnifferGetData);  
  14. if(ret != SUCCESS){  
  15.     printf("启动并口读数据失败!\n");  
  16.     return;  
  17. }else{  
  18.     printf("启动并口读数据成功!\n");  
  19. }

SNIFFER_WriteData

描述：

将数据通过并口发送出去，数据发送频率为SNIFFER_Init函数的SampleRateHz参数设置值。

原型：

  1. int WINAPI SNIFFER_WriteData(int DevHandle,unsigned char *pWriteData,int WriteLen);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

pWriteData  待发送的数据存储缓冲区指针。

WriteLen  待发送的数据字节数。

返回值：

并口写数据状态，0-写数据成功，其他值，写数据失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2sniffer.h"  
  3.    
  4. int DataNum;  
  5.    
  6. int  ret;  
  7. unsigned char WriteBuffer[10240];  
  8. for(int i=0;i<sizeof(WriteBuffer);i++){  
  9.     WriteBuffer[i] = i;  
  10. }  
  11. //配置sniffer  
  12. ret = SNIFFER_Init(DevHandle,1,20000000);  
  13. if(ret != SNIFFER_SUCCESS){  
  14.     printf("Init sniffer error!\n");  
  15.     return 0;  
  16. }  
  17. //并口写数据  
  18. ret = SNIFFER_WriteData(DevHandle,WriteBuffer,sizeof(WriteBuffer));  
  19. if(ret != SUCCESS){  
  20.     printf("Sniffer write error!\n");  
  21.     return 0;  
  22. }else{  
  23.     printf("Sniffer write success!\n");  
  24. }

SNIFFER_ContinueWriteData

描述：

并口连续模式写数据，调用该函数后，数据存放到USB2XXX内部数据缓冲区，然后数据通过并口循环发送出去，直到调用停止写数据函数为止。数据发送频率为SNIFFER_Init函数中的SampleRateHz参数值。

原型：

  1. int WINAPI SNIFFER_ContinueWriteData(int DevHandle,unsigned char *pWriteData,int WriteLen);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

pWriteData  待发送的数据存储缓冲区指针。

WriteLen  待发送的数据字节数。

返回值：

并口写数据状态，0-写数据成功，其他值，写数据失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2sniffer.h"  
  3.    
  4. int DataNum;  
  5.    
  6. int  ret;  
  7. unsigned char WriteBuffer[10240];  
  8. for(int i=0;i<sizeof(WriteBuffer);i++){  
  9.     WriteBuffer[i] = i;  
  10. }  
  11. //配置sniffer  
  12. ret = SNIFFER_Init(DevHandle,1,20000000,8);  
  13. if(ret != SNIFFER_SUCCESS){  
  14.     printf("Init sniffer error!\n");  
  15.     return 0;  
  16. }  
  17. //并口连续写数据，并口将缓冲区中的数据循环发送  
  18. ret = SNIFFER_ContinueWriteData(DevHandle,WriteBuffer,sizeof(WriteBuffer));  
  19. if(ret != SNIFFER_SUCCESS){  
  20.     printf("Sniffer continue write error!\n");  
  21.     return 0;  
  22. }else{  
  23.     printf("Sniffer continue write success!\n");  
  24. }
