概述

USB2XXX有1路CAN总线，P3引脚为CAN_TX，P2为CAN_RX，板子上这两个引脚输出的是TTL电平,需要外接CAN收发器才能接入CAN总线，常用的CAN收发器芯片有TJA1050,SN65HVD230,AU5790(单线CAN)等。

USB2CAN/LIN适配器有2路CAN和1路LIN，内部集成收发器，可以直接接入CAN总线使用。

CAN_Init

描述：

初始化配置USB2CAN适配器，主要是初始化配置CAN控制器，配置CAN总线波特率。

原型：

  1. int WINAPI CAN_Init(int DevHandle, unsigned char CANIndex, PCAN_INIT_CONFIG pCanConfig);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，USB2XXX只能设置为0，USB2CAN/LIN适配器可以设置为0或者1。

pCanConfig CAN控制器初始化配置参数结构体指针，结构体成员取值说明见概述。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. #include <stdlib.h>  
  2. #include <stdio.h>  
  3. #include <string.h>  
  4. #include "usb_device.h"  
  5. #include "usb2can.h"  
  6.   
  7. int CANIndex = 0;  
  8. int ret;  
  9.    
  10. //初始化配置CAN  
  11. CAN_INIT_CONFIG CANConfig;  
  12. #if CAN_MODE_LOOPBACK  
  13. CANConfig.CAN_Mode = 1;//环回模式  
  14. #else  
  15. CANConfig.CAN_Mode = 0;//正常模式  
  16. #endif  
  17. CANConfig.CAN_ABOM = 0;//禁止自动离线  
  18. CANConfig.CAN_NART = 1;//禁止报文重传  
  19. CANConfig.CAN_RFLM = 0;//FIFO满之后覆盖旧报文  
  20. CANConfig.CAN_TXFP = 1;//发送请求决定发送顺序  
  21. //配置波特率,波特率 = 100M/(BRP*(SJW+BS1+BS2))  
  22. CANConfig.CAN_BRP = 25;  
  23. CANConfig.CAN_BS1 = 2;  
  24. CANConfig.CAN_BS2 = 1;  
  25. CANConfig.CAN_SJW = 1;  
  26. ret = CAN_Init(DevHandle,CANIndex,&CANConfig);  
  27. if(ret != CAN_SUCCESS){  
  28.     printf("Config CAN failed!\n");  
  29.     return 0;  
  30. }else{  
  31.     printf("Config CAN Success!\n");  
  32. }  

CAN_Filter_Init

描述：

配置CAN总线接收过滤器，注意，初始化配置CAN后，所有过滤器都使能并接收所有数据，若要使能数据过滤功能，应该调用此函数配置每一个过滤器。

原型：

  1. int WINAPI CAN_Filter_Init(int DevHandle, unsigned char CANIndex, PCAN_FILTER_CONFIG pFilterConfig);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，USB2XXX只能设置为0，USB2CAN/LIN适配器可以设置为0或者1。

pFilterConfig CAN控制器初始化配置参数结构体指针，结构体成员取值说明见概述。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. #include <stdlib.h>  
  2. #include <stdio.h>  
  3. #include <string.h>  
  4. #include "usb_device.h"  
  5. #include "usb2can.h"  
  6.    
  7. int CANIndex = 0;  
  8. int ret;  
  9.   
  10. CAN_FILTER_CONFIG CANFilter;  
  11. CANFilter.FilterIndex = 0;  
  12. CANFilter.Enable = 1;  
  13. CANFilter.ExtFrame = 1;  //需要过滤接收的帧为扩展帧  
  14. CANFilter.FilterMode = 0;//屏蔽位模式  
  15. CANFilter.MASK_IDE = 1; //需要对比接收到的数据IDE位，若跟CANFilter.ID_IDE设置值一致则过滤通过  
  16. CANFilter.MASK_RTR = 0;//无需对比接收到的数据RTR位  
  17. CANFilter.MASK_Std_Ext = 0x0F;//需要对比接收到的ID低4位，且低4位跟CANFilter.ID_Std_Ext低4位相同则过滤通过  
  18. CANFilter.ID_IDE = 1;  
  19. CANFilter.ID_RTR = 0;  
  20. CANFilter.ID_Std_Ext = 0x05;  
  21. CAN_Filter_Init(DevHandle,CANIndex,&CANFilter);  
  22.   
  23. CANFilter.FilterIndex = 1;  
  24. CANFilter.Enable = 1;  
  25. CANFilter.ExtFrame = 1;  //需要过滤接收的帧为扩展帧  
  26. CANFilter.FilterMode = 1;//标识符列表模式  
  27. CANFilter.ID_IDE = 1;      //接收到的数据帧IDE必须为1，也就是必须为扩展帧数据  
  28. CANFilter.ID_RTR = 0;      //接收到的帧类型必须为数据帧  
  29. CANFilter.ID_Std_Ext = 0x12A;//接收到的帧ID必须等于CANFilter.ID_Std_Ext设置的值  
  30. CAN_Filter_Init(DevHandle,CANIndex,&CANFilter);

CAN_SendMsg

描述：

向CAN总线发送CAN消息。

原型：

  1. int WINAPI CAN_SendMsg(int DevHandle, unsigned char CANIndex, PCAN_MSG pCanSendMsg,unsigned int SendMsgNum);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，USB2XXX只能设置为0，USB2CAN/LIN适配器可以设置为0或者1。

pCanSendMsg CAN消息存储缓冲区首地址。

SendMsgNum 即将向CAN总线发送的CAN消息帧数。

返回值：

大于等于0表示成功发送的消息帧数，若小于0则该函数调用失败。

示例：

  1. #include <stdlib.h>  
  2. #include <stdio.h>  
  3. #include <string.h>  
  4. #include "usb_device.h"  
  5. #include "usb2can.h"  
  6.    
  7. int CANIndex = 0;  
  8. int ret;  
  9. //CAN帧数组  
  10. CAN_MSG CanMsg[5];  
  11. for(int i=0;i<5;i++){  
  12.     CanMsg[i].ExternFlag = 0;  
  13.     CanMsg[i].RemoteFlag = 0;  
  14.     CanMsg[i].ID = i;  
  15.     CanMsg[i].DataLen = 8;  
  16.     for(int j=0;j<CanMsg[i].DataLen;j++){  
  17.         CanMsg[i].Data[j] = j;  
  18.     }  
  19. }  
  20. int SendedNum = CAN_SendMsg(DevHandle,CANIndex,CanMsg,5);  
  21. if(SendedNum >= 0){  
  22.     printf("Success send frames:%d\n",SendedNum);  
  23. }else{  
  24.     printf("Send CAN data failed!\n");  
  25. }  

CAN_GetMsg

描述：

从USB2CAN适配器获取已经接收到的CAN消息。

原型：

  1. int WINAPI CAN_GetMsg(int DevHandle, unsigned char CANIndex, PCAN_MSG pCanGetMsg);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，USB2XXX只能设置为0，USB2CAN/LIN适配器可以设置为0或者1。

pCanGetMsg 存储CAN消息缓冲区首地址,注意，为保证接收的数据不会溢出缓冲区，缓冲区大小设置最小为2048帧较为合适。

返回值：

大于等于0表示从CAN适配器内部成功读取到的CAN消息帧数，若返回值小于0则说明调用该函数失败。

示例：

  1. #include <stdlib.h>  
  2. #include <stdio.h>  
  3. #include <string.h>  
  4. #include "usb_device.h"  
  5. #include "usb2can.h"  
  6.    
  7. int CANIndex = 0;  
  8. int ret;  
  9. //CAN消息帧存储缓冲区  
  10. CAN_MSG CanMsgBuffer[10240];  
  11. int CanNum = CAN_GetMsg(DevHandle,CANIndex,CanMsgBuffer);  
  12. if(CanNum > 0){  
  13.     printf("CanNum = %d\n",CanNum);  
  14.     for(int i=0;i<CanNum;i++){  
  15.         printf("CanMsg[%d].ID = %d\n",i,CanMsgBuffer[i].ID);  
  16.         printf("CanMsg[%d].TimeStamp = %d\n",i,CanMsgBuffer[i].TimeStamp);  
  17.         printf("CanMsg[%d].Data = ",i);  
  18.         for(int j=0;j<CanMsgBuffer[i].DataLen;j++){  
  19.             printf("%02X ",CanMsgBuffer[i].Data[j]);  
  20.         }  
  21.         printf("\n");  
  22.     }  
  23. }else if(CanNum == 0){  
  24.     printf("No CAN data!\n");  
  25. }else{  
  26.     printf("Get CAN data error!\n");  
  27. }

CAN_SetSchedule

描述：

配置CAN调度表数据，该数据是放到适配器内部，被适配器内部调度表定时器调度运行，其帧发送间隔时间精度更高，一般误差在1ms以内。

原型：

int WINAPI CAN_SetSchedule(int DevHandle, unsigned char CANIndex, PCAN_MSG pCanMsgTab,unsigned char *pMsgNum,unsigned short *pSendTimes,unsigned char MsgTabNum); 

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，可以设置为0或者1。

pCanMsgTab CAN调度表数据指针。

pMsgNum 每个调度表里面包含有效帧数指针。

pSendTimes 每个调度表里面帧发送次数，若为0xFFFF，则循环发送，通过调用CAN_StopSchedule函数停止发送。

MsgTabNum 调度表数量。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. //准备CAN调度表数据  
  2. const int MsgNum = 20;  
  3. CAN_MSG CanMsg[MsgNum];  
  4. for(int i=0;i<MsgNum;i++){  
  5.     CanMsg[i].ExternFlag = 0;  
  6.     CanMsg[i].RemoteFlag = 0;  
  7.     CanMsg[i].ID = i+1;  
  8.     CanMsg[i].DataLen = 8;  
  9.     CanMsg[i].TimeStamp = 20;//帧周期为20ms，必须设置  
  10.     for(int j=0;j<CanMsg[i].DataLen;j++){  
  11.         CanMsg[i].Data[j] = j;  
  12.     }  
  13.     CanMsg[i].Data[0] = i+1;  
  14. }  
  15. //总共3个调度表，第一个表里面包含3帧数据，第二个调度表包含6帧数据，第三个调度表包含11帧数据  
  16. uint8_t MsgTabNum[3]={3,6,11};  
  17. //第一个调度表循环发送数据，第二个调度表循环发送数据，第三个调度表只发送3次  
  18. uint16_t SendTimes[3]={0xFFFF,0xFFFF,3};  
  19. ret = CAN_SetSchedule(DevHandle[0],0,CanMsg,MsgTabNum,SendTimes,3);  
  20. if(ret == CAN_SUCCESS){  
  21.     printf("Set CAN Schedule Success\n");  
  22. }else{  
  23.     printf("Set CAN Schedule Error ret = %d\n", ret);  
  24.     return 0;  
  25. }  

CAN_StartSchedule

描述：

启动CAN调度表，开始发送CAN数据，每次只能启动一个调度表，在启动新的调度表之前，需要把已经启动的调度表停止。

原型：

  1. int WINAPI CAN_StartSchedule(int DevHandle, unsigned char CANIndex, unsigned char MsgTabIndex,unsigned char TimePrecMs,unsigned char OrderSend);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，USB2XXX只能设置为0，USB2CAN/LIN适配器可以设置为0或者1。

MsgTabIndex 调度表索引号，该索引号不能大于调度表数量。

TimePrecMs 调度表执行定时参数，比如调度表中，帧周期最小为10ms，那么此处可以设置为10，该值不能设置小于5.

OrderSend 0-调度表里面数据并行发送，1-调度表里面数据顺序发送。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. ret = CAN_StartSchedule(DevHandle[0],SendCANIndex,0,10,0);//启动第一个调度表  
  2. if(ret == CAN_SUCCESS){  
  3.     printf("Start CAN Schedule 1 Success\n");  
  4. }else{  
  5.     printf("Start CAN Schedule 1 Error ret = %d\n", ret);  
  6.     return 0;  
  7. }
