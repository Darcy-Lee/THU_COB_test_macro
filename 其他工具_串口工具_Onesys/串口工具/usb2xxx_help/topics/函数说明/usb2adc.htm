概述 

本文主要介绍关于USB2XXX的ADC操作相关API，在调用这些函数前需要调用设备操作相关函数，比如扫描设备，打开设备等，详情请查看《设备操作相关函数说明 》。

下面是本文函数可能会用到的一些宏定义：

  1. //定义函数返回错误代码  
  2. #define ADC_SUCCESS                          (0)   //函数执行成功  
  3. #define ADC_ERR_NOT_SUPPORT       (-1)  //适配器不支持该函数  
  4. #define ADC_ERR_USB_WRITE_FAIL  (-2)  //USB写数据失败  
  5. #define ADC_ERR_USB_READ_FAIL    (-3)  //USB读数据失败  
  6. #define ADC_ERR_CMD_FAIL               (-4)  //命令执行失败  
  7. #define ADC_ERR_CH_NO_INIT          (-5)  //该通道未初始化  
  8.   
  9. //定义连续采集数据模式下的回调函数  
  10. typedef  int (WINAPI *PADC_GET_DATA_HANDLE)(int DevHandle,unsigned short *pData,int DataNum);//接收数据回掉函数  

ADC_Init

描述：

初始化配置ADC，调用该函数之后可以通过调用ADC_Read函数获取ADC的值。

原型：

  1. int WINAPI ADC_Init(int DevHandle,char Channel,int SampleRateHz);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数或得。

Channel  需要初始化配置的ADC通道，每个bit位代表一个通道，最低位代表通道0，最多4个通道。对应通道值为1则初始化并使能该通道ADC。在调用ADC_Read函数的时候，会根据该通道值返回对应的数据。

SampleRateHz  ADC采集多个数据时的采样频率，单位为Hz，最大2500000Hz。若同时使能多个通道时，其最大采样率会根据通道值降低，比如同时使能了4个通道，那么最大采样率为2500000/4=625000.

返回值：

初始化ADC的状态，0-配置成功，其他值，配置失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2adc.h"  
  3.    
  4. int ret;  
  5. char ADC_Channel = 0x01;//初始化配置ADC_IN0 
  6. ret = ADC_Init(DevHandle,ADC_Channel,1000000);  
  7. if(ret != SUCCESS){  
  8.     printf("初始化配置ADC失败!\n");  
  9.     return;  
  10. }  

ADC_Read

描述：

立即获取ADC采样值，其通道通过ADC_Init函数指定。

原型：

  1. int WINAPI ADC_Read(int DevHandle,short *pData,int DataNum);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

pData  数据存储缓冲区首地址，当只初始化了1路ADC时，其数据只有DataNum个，若初始化了n路ADC，则返回的数据有n*DataNum个数据，数据按照ADC_IN0,ADC_IN1,…ADC_INn这样依次排列，假如说初始化配置了ADC_IN0,ADC_IN1,ADC_IN2,DataNum为3，那么总的数据有3*3=9个，数据的顺序为ADC_IN0_DATA0,ADC_IN1_DATA0,ADC_IN2_DATA0,ADC_IN0_DATA1,ADC_IN1_DATA1,ADC_IN2_DATA1,ADC_IN0_DATA2,ADC_IN1_DATA2,ADC_IN2_DATA2.

DataNum  每个ADC通道采集的数据数。

返回值：

获取ADC数据的状态，0-获取数据成功，其他值，获取数据失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2adc.h"  
  3.    
  4. int ret;  
  5. int DataNum = 10;        //每通道获取10个ADC采样值  
  6. short Buffer[DataNum*BitCount(ADC_Channel)];   
  8. ret = ADC_Read(DevHandle,Buffer,DataNum);  
  9. if(ret != SUCCESS){  
  10.     printf("Read adc error!\n");  
  11.     return 0;  
  12. }else{  
  13.     for(int i=0;i<DataNum*BitCount(ADC_Channel);i++){  
  14.         printf("ADC Data[%d] = %fV\n",i,(Buffer[i]*3.3)/4095);  
  15.     }  
  16. }  

ADC_StartContinueRead

描述：

启动ADC连续采样，调用该函数后，ADC将以设置的采样率连续采集数据，直到调用ADC_StopContinueRead函数为止。可以通过传入的回调函数或者调用ADC_GetData函数获取采集到的数据。

原型：

  1. int WINAPI ADC_StartContinueRead(int DevHandle,char Channel,int SampleRateHz,int FrameSize,PADC_GET_DATA_HANDLE pGetDataHandle); 

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

Channel  需要初始化配置的ADC通道，每个bit位代表一个通道，最低位代表通道0，最多4个通道。对应通道值为1则初始化并使能该通道ADC。在调用ADC_Read函数的时候，会根据该通道值返回对应的数据。

SampleRateHz  ADC采集多个数据时的采样频率，单位为Hz，最大2500000Hz。若同时使能多个通道时，其最大采样率会根据通道值降低，比如同时使能了4个通道，那么最大采样率为2500000/4=625000.

FrameSize  连续采样时每次传输数据个数SampleRateHz*1000/FrameSize最好大于或者等于20。

pGetDataHandle  读取到数据之后的回调函数，该函数需要上层代码自己实现，并在里面做对应的数据处理。

返回值：

启动连续采样的状态，0-启动成功，其他值，启动失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2adc.h"  
  3.    
  4. int GetAdcDataHandle(int DevHandle,unsigned short *pData,int DataNum)  
  5. {  
  6.     printf("Get %d Byte Data\n",DataNum);  
  7.     printf("ADC Data = %fV\n",(pData[0]*3.3)/4095);  
  8. }  
  9.    
  10. int ret;  
  11. char ADC_Channel = 0x01;  
  12. ret = ADC_StartContinueRead(DevHandle,ADC_Channel,1000000,GetAdcDataHandle);  
  13. if(ret != ADC_SUCCESS){  
  14.     printf("Start Continue Read adc error!\n");  
  15.     return 0;  
  16. }else{  
  17.     printf("Start Continue Read ADC ADC_SUCCESS!\n");  
  18. }  

ADC_StopContinueRead

描述：

停止ADC的连续采样。

原型：

  1. int WINAPI ADC_StopContinueRead(int DevHandle);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

返回值：

停止ADC连续采样状态，0-停止成功，其他值，停止失败。

示例：

  1. #include <stdio.h>  
  2. #include "usb2adc.h"  
  3.    
  4. int ret;  
  5. ret = ADC_StopContinueRead(DevHandle);  
  6. if(ret != SUCCESS){  
  7.     printf("停止ADC连续采样失败!\n");  
  8.     return;  
  9. }  

ADC_GetData

描述：

启动ADC连续采集数据之后，通过该函数获取存入数据缓冲区中的数据，该函数应在ADC_StopContinueRead函数被调用之前连续调用，以防止接收的数据太大，导致数据缓冲区溢出。若在启动连续获取数据函数时传入了回调函数，则无需通过该函数来获取数据。

原型：

  1. int WINAPI ADC_GetData(int DevHandle,unsigned short *pDataBuffer,int BufferSize);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

pDataBuffer  数据存储缓冲区首地址。缓冲区最小为10240个unsigned short大小，其他缓冲区大小最好按照10240整数倍分配。

BufferSize  数据缓冲区大小，单位为short，比如数据缓冲区声明为unsigned short DataBuffer[10240],那么该值为10240.

返回值：

从数据接收缓冲区实际获取到的数据数，单位为unsigned short数据类型大小。

示例：

  1. #include <stdio.h>  
  2. #include "usb2adc.h"  
  3.    
  4. unsigned short ReadDataBuffer[20480];  
  5. int ReadDataNum;  
  6. ReadDataNum = ADC_GetData(DevHandle,ReadDataBuffer,20480);  
  7. printf("获取到%d个数据\n",ReadDataNum );
