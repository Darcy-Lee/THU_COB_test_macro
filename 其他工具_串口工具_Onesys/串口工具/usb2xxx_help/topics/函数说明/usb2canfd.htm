概述

UTA0503才支持CANFD功能，若需要使用CANFD功能，请注意使用适配器的型号。该版本的适配器同时支持发送CAN2.0A/B数据。

CANFD_Init

描述：

初始化配置USB2CANFD适配器，主要是初始化配置CAN控制器，配置CAN总线波特率。

原型：

int WINAPI CANFD_Init(int DevHandle, unsigned char CANIndex, CANFD_INIT_CONFIG *pCanConfig);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，0-对应CAN1,1-对应CAN2。

pCanConfig CAN控制器初始化配置参数结构体指针，结构体成员取值说明见概述。

返回值：

函数执行状态，为CANFD_SUCCESS表示成功，否则为失败。

示例：

  1. //初始化配置CAN  
  2. CANFD_INIT_CONFIG CANFDConfig;  
  3. CANFDConfig.Mode = 1;        //0-正常模式，1-自发自收模式  
  4. CANFDConfig.RetrySend = 1;   //使能自动重传  
  5. CANFDConfig.ISOCRCEnable = 1;//使能ISOCRC  
  6. CANFDConfig.ResEnable = 1;   //使能内部终端电阻（若总线上没有终端电阻，则必须使能终端电阻才能正常传输数据）  
  7. //波特率参数可以用TCANLINPro软件里面的波特率计算工具计算  
  8. //仲裁段波特率参数,波特率=40M/NBT_BRP*(1+NBT_SEG1+NBT_SEG2)  
  9. CANFDConfig.NBT_BRP = 1;  
  10. CANFDConfig.NBT_SEG1 = 63;  
  11. CANFDConfig.NBT_SEG2 = 16;  
  12. CANFDConfig.NBT_SJW = 16;  
  13. //数据域波特率参数,波特率=40M/DBT_BRP*(1+DBT_SEG1+DBT_SEG2)  
  14. CANFDConfig.DBT_BRP = 1;  
  15. CANFDConfig.DBT_SEG1 = 15;  
  16. CANFDConfig.DBT_SEG2 = 4;  
  17. CANFDConfig.DBT_SJW = 4;  
  18. ret = CANFD_Init(DevHandle[0],CANIndex,&CANFDConfig);  
  19. if(ret != CANFD_SUCCESS){  
  20.     printf("CANFD Init Error!\n");  
  21.     return 0;  
  22. }else{  
  23.     printf("CANFD Init Success!\n");  
  24. }  

CANFD_SetFilter

描述：

配置CAN总线接收过滤器。注意，初始化配置CAN后，所有过滤器都使能并接收所有数据，若要使能数据过滤功能，应该调用此函数配置每一个过滤器。

原型：

int WINAPI CANFD_SetFilter(int DevHandle, unsigned char CANIndex, CANFD_FILTER_CONFIG *pCanFilter,unsigned char Len);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，0-对应CAN1,1-对应CAN2。

pCanFilter CAN控制器初始化配置参数结构体指针，结构体成员取值说明见概述。

Len 待配置的过滤器个数。

返回值：

函数执行状态，为CANFD_SUCCESS表示成功，否则为失败。

示例：

  1. //配置过滤器，若不配置过滤器，默认是接收所有数据  
  2. CANFD_FILTER_CONFIG CANFDFilter;  
  3. CANFDFilter.Index = 0;//取值范围为：0~31  
  4. CANFDFilter.Enable = 1;//0-禁止，1-使能  
  5. //配置为只接收扩展帧数据  
  6. CANFDFilter.ID_Accept = 0x80000000;  
  7. CANFDFilter.ID_Mask = 0x80000000;  
  8. ret = CANFD_SetFilter(DevHandle[0], CANIndex, &CANFDFilter, 1);  
  9. if (ret != CANFD_SUCCESS){  
  10.     printf("Config filter failed!\n");  
  11.     return 0;  
  12. }else{  
  13.     printf("Config filter success!\n");  
  14. }

CANFD_SendMsg

描述：

向CAN总线发送CAN消息。

原型：

int WINAPI CANFD_SendMsg(int DevHandle, unsigned char CANIndex, CANFD_MSG *pCanSendMsg,unsigned int SendMsgNum); 

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，0-对应CAN1,1-对应CAN2。

pCanSendMsg CAN消息存储缓冲区首地址。

SendMsgNum 即将向CAN总线发送的CAN消息帧数。

返回值：

大于等于0表示成功发送的消息帧数，若小于0则该函数调用失败。

示例：

  1. //发送CAN数据  
  2. CANFD_MSG CanMsg[5];  
  3. for (int i = 0; i < 5; i++){  
  4.     CanMsg[i].Flags = 0;//bit[0]-BRS,bit[1]-ESI,bit[2]-FDF,bit[6..5]-Channel,bit[7]-RXD  
  5.     CanMsg[i].DLC = 8;  
  6.     CanMsg[i].ID = i|CANFD_MSG_FLAG_IDE;  
  7.     for (int j = 0; j < CanMsg[i].DLC; j++){  
  8.         CanMsg[i].Data[j] = j;  
  9.     }  
  10. }  
  11. int SendedMsgNum = CANFD_SendMsg(DevHandle[0], CANIndex, CanMsg, 5);  
  12. if (SendedMsgNum >= 0){  
  13.     printf("Success send frames:%d\n", SendedMsgNum);  
  14. }else{  
  15.     printf("Send CAN data failed!\n");  
  16.     return 0;  
  17. }  

CANFD_GetMsg

描述：

读取接收到的CAN消息。

原型：

int WINAPI CANFD_GetMsg(int DevHandle, unsigned char CANIndex, CANFD_MSG *pCanGetMsg,int BufferSize);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，0-对应CAN1,1-对应CAN2。

pCanGetMsg 存储CAN消息缓冲区首地址。

BufferSize 存储CAN消息缓冲区大小。

返回值：

大于等于0表示从CAN适配器内部成功读取到的CAN消息帧数，若返回值小于0则说明调用该函数失败。

示例：

  1. //读取接收数据缓冲中的数据  
  2. CANFD_MSG CanMsgBuffer[1024];  
  3. int GetMsgNum = CANFD_GetMsg(DevHandle[0], CANIndex, CanMsgBuffer, 1024);  
  4. if (GetMsgNum > 0){  
  5.     for (int i = 0; i < GetMsgNum; i++){  
  6.         printf("CanMsg[%d].ID = 0x%08X\n", i, CanMsgBuffer[i].ID & CANFD_MSG_FLAG_ID_MASK);  
  7.         printf("CanMsg[%d].TimeStamp = %d\n", i, CanMsgBuffer[i].TimeStamp);  
  8.         printf("CanMsg[%d].Data = ", i);  
  9.         for (int j = 0; j < CanMsgBuffer[i].DLC; j++)  
  10.         {  
  11.             printf("0x%02X ", CanMsgBuffer[i].Data[j]);  
  12.         }  
  13.         printf("\n");  
  14.     }  
  15. }else if (GetMsgNum < 0){  
  16.     printf("Get CAN data error!\n");  
  17. }

CANFD_SetSchedule

描述：

配置CAN调度表数据，该数据是放到适配器内部，被适配器内部调度表定时器调度运行，其帧发送间隔时间精度更高，一般误差在1ms以内。

原型：

int WINAPI CANFD_SetSchedule(int DevHandle, unsigned char CANIndex, CANFD_MSG pCanMsgTab,unsigned char *pMsgNum,unsigned short *pSendTimes,unsigned char MsgTabNum); 

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，可以设置为0或者1。

pCanMsgTab CAN调度表数据指针。

pMsgNum 每个调度表里面包含有效帧数指针。

pSendTimes 每个调度表里面帧发送次数，若为0xFFFF，则循环发送，通过调用CAN_StopSchedule函数停止发送。

MsgTabNum 调度表数量。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. //准备CAN调度表数据  
  2. const int AllMsgNum = 20;  
  3. CANFD_MSG CanMsg[AllMsgNum];  
  4. for (int i = 0; i < AllMsgNum; i++){  
  5.     CanMsg[i].Flags = 0;//bit[0]-BRS,bit[1]-ESI,bit[2]-FDF,bit[6..5]-Channel,bit[7]-RXD  
  6.     CanMsg[i].DLC = 8;  
  7.     CanMsg[i].ID = i|CANFD_MSG_FLAG_IDE;  
  8.     for (int j = 0; j < CanMsg[i].DLC; j++){  
  9.         CanMsg[i].Data[j] = j;  
  10.     }  
  11.     CanMsg[i].Data[0] = i;  
  12.     CanMsg[i].TimeStamp = 20;//每帧间隔10ms  
  13. }  
  14. //总共3个调度表，第一个表里面包含3帧数据，第二个调度表包含6帧数据，第三个调度表包含11帧数据  
  15. uint8_t MsgTabNum[3]={3,6,11};  
  16. //第一个调度表循环发送数据，第二个调度表循环发送数据，第三个调度表只发送3次

CANFD_StartSchedule

描述：

启动CAN调度表，开始发送CAN数据，每次只能启动一个调度表，在启动新的调度表之前，需要把已经启动的调度表停止。

原型：

  1. int WINAPI CANFD_StartSchedule(int DevHandle, unsigned char CANIndex, unsigned char MsgTabIndex,unsigned char TimePrecMs,unsigned char OrderSend);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，可以设置为0或者1。

MsgTabIndex 调度表索引号，该索引号不能大于调度表数量。

TimePrecMs 调度表执行定时参数，比如调度表中，帧周期最小为10ms，那么此处可以设置为10，该值不能设置小于5.

OrderSend 0-调度表里面数据并行发送，1-调度表里面数据顺序发送。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。

示例：

  1. ret = CANFD_StartSchedule(DevHandle[0],SendCANIndex,0,10,0);//启动第一个调度表  
  2. if(ret == CAN_SUCCESS){  
  3.     printf("Start CAN Schedule 1 Success\n");  
  4. }else{  
  5.     printf("Start CAN Schedule 1 Error ret = %d\n", ret);  
  6.     return 0;  
  7. }  

CANFD_StopSchedule 

描述：

停止已经启动的调度表。

原型：

  1. int WINAPI CANFD_StopSchedule(int DevHandle, unsigned char CANIndex);  

参数：

DevHandle 设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice函数获得。

CANIndex CAN通道索引号，可以设置为0或者1。

返回值：

函数执行状态，为CAN_SUCCESS表示成功，否则为失败。
