概述

CAN_BOOT_Init

描述：

初始化配置CAN Bootloader接口。

原型：

  1. int WINAPI CAN_BOOT_Init(int DevHandle,int CANIndex,unsigned int SendDataID,unsigned int ReceiveDataID,unsigned char IDType,int BaudRate);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

SendDataID   CAN主节点（USB2CAN适配器）向从节点发送数据的帧ID，可以是标准ID或者扩展ID。

ReceiveDataID  CAN主节点（USB2CAN适配器）接收从节点返回数据的帧ID，可以是标准ID或者扩展ID。

IDType  帧ID类型，0-标准帧，1-扩展帧

BaudRate CAN总线波特率值，单位为Hz，比如要配置为500K，则传入500000。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

CAN_BOOT_GetFWInfo

描述：

获取节点固件信息，

原型：

  1. int WINAPI CAN_BOOT_GetFWInfo(int DevHandle,int CANIndex,unsigned char NAD,unsigned char *pFWType,unsigned int *pFWVersion,unsigned int TimeOutOfMs);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

NAD 节点地址，取值范围0x01~0x7E。

pFWType  获取到的节点固件类型，0x55-当前节点固件为Bootloader固件，0xAA-当前固件为App固件。

pFWVersion  固件版本号，比如值为0x0102030A，那么固件的版本为V1.2.3.10。

TimeOutOfMs  适配器发送获取节点固件信息后，等待从节点返回信息的时间，超过这个时间后函数返回超时错误。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

示例：

CAN_BOOT_EraseApp

描述：

擦除App区域的固件数据，为后面App数据的烧入做准备。

原型：


int WINAPI CAN_BOOT_EraseApp(int DevHandle,int CANIndex,unsigned char NAD,unsigned int AppSizeOfByte,unsigned int TimeOutOfMs); 

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用 USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

NAD 节点地址，取值范围0x01~0x7E。

AppSizeOfByte  需要擦除的App区域大小，单位为字节。

TimeOutOfMs  发送固件擦除指令后，等待从节点返回状态信息的超时时间，一般内部存储区域擦除都比较耗时，所以这个超时时间可能得根据实际的情况设置大一点。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

示例： 

CAN_BOOT_SetAddrOffset

描述：

设置后续写入App数据的起始偏移地址。并获取从节点每次能接收App数据字节数大小。

原型：


int WINAPI CAN_BOOT_SetAddrOffset(int DevHandle,int CANIndex,unsigned char NAD,unsigned int AddrOffset,unsigned short *pBufferSize,unsigned int TimeOutOfMs);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用 USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

NAD 节点地址，取值范围0x01~0x7E。

AddrOffset  准备写入App数据的偏移地址，注意这个不是芯片内部的实际数据存储地址，是App数据分包传输的偏移地址。

pBufferSize  从节点单次能接收的最大App数据字节数，比如为1024，那么每次传输的数据就不能大于1024。

TimeOutOfMs  发送固件擦除指令后，等待从节点返回状态信息的超时时间，一般内部存储区域擦除都比较耗时，所以这个超时时间可能得根据实际的情况设置大一点。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

示例： 

CAN_BOOT_SendAppData

描述：

将App数据发送到从节点内部数据缓冲区中。

原型：

  1. int WINAPI CAN_BOOT_SendAppData(int DevHandle,int CANIndex,unsigned char NAD,unsigned char *pData,unsigned short DataLen,unsigned short *pCRC16);  

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用 USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

NAD 节点地址，取值范围0x01~0x7E。

pData  待发送的数据缓冲区指针。

DataLen  待发送的数据字节数，注意不能大于从节点能接收的最大字节数。

pCRC16  这些数据的CRC16值，用于后续数据写入时校验。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

示例： 

CAN_BOOT_WriteAppData

描述：

将从节点内部数据缓冲区中的数据写入芯片中的程序存储器中，并对数据进行校验。

原型：

int WINAPI CAN_BOOT_WriteAppData(int DevHandle,int CANIndex,unsigned char NAD,unsigned short CRC16,unsigned int TimeOutOfMs); 

参数：

DevHandle  设备句柄，本质为设备序号的低4字节，可以通过调用USB_ScanDevice 函数获得。

CANIndex  CAN通道号，取值0或者1。

NAD 节点地址，取值范围0x01~0x7E。

CRC16  从节点内部数据缓冲区中数据的CRC16值，可以通过CAN_BOOT_SendAppData函数获取到。

TimeOutOfMs  发送该指令后，等待从节点返回状态数据的超时时间，由于固件写入可能会比较耗时，所以这个超时时间可能得根据实际的情况设置大一点。

返回值：

函数执行状态，0-函数执行成功，若小于0则该函数调用失败。

示例：
